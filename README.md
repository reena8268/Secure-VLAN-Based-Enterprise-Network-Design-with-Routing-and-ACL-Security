# 🔐 Secure VLAN-Based Enterprise Network with Branch Connectivity, Firewall Security, and Server Services

## 📖 Overview
This project presents the design and simulation of a secure enterprise network using Cisco Packet Tracer. The network is divided into multiple VLANs representing different departments to improve security and network performance. Inter-VLAN routing enables controlled communication between departments, while Access Control Lists (ACLs) act as a firewall by preventing unauthorized users from accessing sensitive network resources. The network also provides centralized DHCP, DNS, and Web Server services, along with a Backup Server for service continuity. A branch office is connected to the headquarters network using static routing to simulate a real-world enterprise environment. 2 3

## ⚙️ Working Principle
1. The enterprise network is divided into multiple VLANs for Employees, Students, Management, Administration, and Servers.
2. A Layer 3 router performs Inter-VLAN Routing using Router-on-a-Stick configuration.
3. DHCP dynamically assigns IP addresses to connected devices.
4. DNS resolves domain names for internal users, while the Web Server hosts enterprise services.
5. ACLs function as a firewall by restricting unauthorized VLANs from accessing protected server resources.
6. A Backup Server ensures continuous service availability if the primary server fails.
7. A branch office is connected to the headquarters using static routing, allowing secure communication and access to centralized services. 4 5
